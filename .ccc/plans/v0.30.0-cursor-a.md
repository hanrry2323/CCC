## Phase 2: CPU=0 误杀修复

### 目标

macOS 上 `ps -o %cpu=` 是生命周期均值。运行 >5min 的 LLM 调用纯网络等待时 CPU 接近 0，被`_check_and_mark_hung()`误杀。

### 改动

**文件**：`scripts/ccc-engine.py`
**函数**：`_check_and_mark_hung()`，约 L2168-2176

找到以下代码块：

```python
try:
    cpu = float(result.stdout.strip())
except ValueError:
    engine_log(...)
    continue
if cpu > 0.0:
    continue
```

在其后添加（`if cpu > 0.0: continue` 未命中时的二次确认）：

```python
# v0.30.0: macOS ps %cpu 是生命周期均值，网络等待型进程可能 CPU≈0 但非 hung
# 二次确认：检查 pids 目录下该 subid 附属文件的最后修改时间
_latest = 0.0
try:
    for _pf in sorted(
        pids_dir.iterdir(), key=lambda x: x.stat().st_mtime, reverse=True
    ):
        if _pf.name.startswith(f"{subid}.") and _pf.name != hung_path.name:
            _latest = _pf.stat().st_mtime
            break
except OSError:
    pass
if _latest and (now.timestamp() - _latest) < 120:
    # 最近 2 分钟有文件活动（如 stdout/result 持续写入）→ 进程活着
    continue
```

### 禁止事项

- 不要简单增大 `_HANG_CHECK_INTERVAL_SEC`（300→更大）
- 不要移除 `ps %cpu=` 检查
- 不要改动 patrol-v4.py

# v0.11 计划：CCC 开箱即用调度 + 队列真测试

## 目标
v0.11 = v0.10 评估里的 2 个缺口：
- a: 缺 install-ccc-scheduler + 默认钩子模板（开箱即用）
- b: 缺 ccc-queue 多 phase 真测（验升级 + 暂停恢复）

## Phase

### Phase a1: 默认钩子模板
**产物**: `templates/hooks/post-exec.sh` + `on-error.sh` + `pre-commit.sh`
- post-exec: 自动 `git add` + `git commit -m "phase X done"`（无 commit 留 pending）
- on-error: 自动 L2 桌面通知 + 落 abnormal-reports
- pre-commit: 简单 lint 兜底（grep TODO/FIXME/print 警告）

### Phase a2: 一键安装器
**产物**: `scripts/install-ccc-scheduler.sh`
- 生成 plist 用 `realpath` 推 CCC_HOME（无硬编码）
- plist 含 `opencode-watchdog.sh` 前置调用（红线 X3）
- 三个模式: 默认装 / `--uninstall` / `--dry-run`
- uninstall 必清: launchctl unload + rm plist + 清 pid

### Phase b1: 队列 3 phase 全成功
**产物**: `tests/scripts/test_queue_e2e_3phase_pass.py`
- 用 cmd= 注入 echo 模拟（v0.10 已支持）
- 验 3 phase 顺序跑 + 状态写回 done

### Phase b2: 队列中间失败升级
**产物**: `tests/scripts/test_queue_e2e_mid_fail.py`
- 造中间 phase 失败（exit 非 0）
- 验队列退出 5 + L3 通知触发

### Phase b3: 续跑恢复
**产物**: `tests/scripts/test_queue_e2e_resume.py`
- phases.json 改 status=pending 后重跑
- 验续跑跳到第一个 pending phase

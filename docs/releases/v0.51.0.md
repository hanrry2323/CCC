# Release v0.51.0 — 编排仓与业务仓角色分离（对内 M2）

**Tag:** `v0.51.0` · **Date:** 2026-07-18 · **性质:** 高可维护性：CCC 自消费切断

## 为何

此前两件事搅在一起：

1. CCC 登记在 Engine 消费名单 → evolve/debt 等卡在 CCC 看板上跑（底座开发自己）
2. 平台 bug 习惯进业务仓就地修 → 无统一维护窗口

本版本把 **CCC = orch（编排）**、**业务仓 = app** 锁死：Cursor 改底座一次全局生效；Hub/Engine 只驱动业务仓。

## 角色矩阵

```text
你 + Cursor ──改 CCC 本体──► 全局生效
你 + Hub   ──定稿下达──► Engine ──只扫 apps──► 业务仓看板
Hub        ──运维观察──► CCC（可读控制面，不可下达）
```

## 验收

```bash
cd ~/program/CCC
python3 scripts/ccc-workspace-doctor.py migrate
python3 scripts/ccc-workspace-doctor.py
# 期望：CCC role=orch engine=False；apps≤10

# Hub：选 CCC → 下达 → 应 400 / toast 拒绝
# 有业务仓时默认项目 ≠ ccc
```

# Cursor 模型路由（四面协作）

> 对齐 [`four-role-fluency-charter.md`](four-role-fluency-charter.md)。  
> 目的：少烧高级模型、少让 Auto 做越权设计。

## 一句话

| 窗 | 默认模型 | 何时升级 |
|----|----------|----------|
| **架构** | **高级**（Composer / 高档） | 几乎总是；定 brief、验收、边界冲突 |
| **壳 / 过桥 / 编排** | **Auto** | brief 已 `accepted` 且白名单清晰时 |
| **壳 / 过桥 / 编排** | **高级** | 根因不清、要改契约、跨两面、或 Auto 连续打回两次 |

## 用 Auto

- 按已定稿 brief 改白名单内代码  
- 补文案、接已有 API、填 §8 回贴  
- 跑 `py_compile` / pytest / 既有 smoke  
- 机械重构（改名、抽函数、对齐模板）

## 用高级模型

- 写 / 改 brief、定契约、验收打回理由  
- 首次设计（如本轮 Hub recover loop 这类行为规格）  
- 跨壳+过桥+编排的根因调查  
- 安全 / 幂等 / 状态机易谎报处  
- Auto 已两次偏离 brief 或改到白名单外

## 禁止

- Auto 窗自行扩大范围或改 `docs/briefs` 定稿结论（可填 §8，不可改 §1–§7 目标）  
- 架构窗用 Auto「顺便写完实现」与执行窗抢同一文件（除非执行窗空窗且 brief 写明例外）

## 本流水线默认

工厂模式（用户只粘贴）：架构窗 = 高级；派单正文写明「本任务用 Auto」。

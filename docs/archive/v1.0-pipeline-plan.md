## Loop 策略

修复 B1 后 Engine 会自动：
1. 每 tick 优先处理 in_progress（检查完成）
2. 交替处理 backlog（product_role 拆分 1 个） + planned（dev_role 启动 1 个）
3. 空闲时跑 testing gate（reviewer+tester）
4. Patrol 每 10 分钟巡检一次监控状态

**任务来源**：
- 当前 backlog 已有 80+ 任务（CCC 3 + qxo 15 + xianyu 20 + qb 20 + qx 20）
- 任务消耗完后 Engine 自动补充（通过 audit_role / 反馈回路）
- 也可以注入新需求

## 验收标准

1. **Engine 日志显示 dev_role 被正常启动执行**（之前只有 product_role）
2. **至少 1 个任务从 backlog → planned → in_progress → testing → verified → released**
3. **Patrol 报告 stale=0, warn=0 持续 24h**
4. **所有 workspace 看板显示任务在正常流转**
